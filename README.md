# Project Title
OSS Audit Toolkit  
A set of Bash scripts for Linux system auditing, FOSS inspection, log analysis, and open-source manifesto generation.

## Overview
OSS Audit Toolkit is a lightweight, script-based collection for quickly inspecting a Linux system, validating common FOSS tooling, auditing key directories, and analyzing logs. It also includes an interactive manifesto generator to produce a personalized open-source statement.  
Use cases include Linux onboarding, classroom labs, basic system checks, and open-source advocacy demos.

## Features
- System identity report with distro, kernel, user, uptime, and timestamp.
- Git package inspection with version and binary location.
- Disk and permission audit of core system directories.
- Log analyzer that counts keyword occurrences and shows recent matches.
- Interactive manifesto generator that writes a personalized statement to disk.

## Architecture / How It Works
- The toolkit is a set of standalone Bash scripts; each script runs independently.
- Scripts rely on standard Linux utilities and files (e.g., `/etc/os-release`, `/var/log/syslog`).
- Data flow is simple: each script reads system state or user input, processes it, and prints a formatted report to stdout. The manifesto generator also writes a text file to the current directory.

## Tech Stack
Language: Bash  
Framework: None  
Libraries: GNU coreutils (e.g., `awk`, `sed`, `grep`, `du`, `tail`, `nl`)  
Tools: `uname`, `whoami`, `uptime`, `date`, `git` (optional)


## Usage
Run any script directly with Bash:

```bash
./scripts/script1_system_report.sh
./scripts/script2_package_inspector.sh
./scripts/script3_disk_auditor.sh
./scripts/script4_log_analyzer.sh
./scripts/script5_manifesto_gen.sh
```

Notes:
- `script2_package_inspector.sh` checks for `git` specifically.
- `script4_log_analyzer.sh` reads `/var/log/syslog` and searches for the keyword `error`. On non-Debian systems, update `LOGFILE` inside the script if needed.
- `script5_manifesto_gen.sh` prompts for input and generates `manifesto_<username>_<YYYYMMDD>.txt` in the current directory.
- No CLI flags are implemented; customize behavior by editing the variables at the top of each script where applicable.

## Environment Variables
None required. Scripts read standard system state and use built-in utilities.
